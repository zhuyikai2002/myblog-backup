现实与理想的博弈：我为什么阶段性 All-in Python

发表于2026-01-22|更新于2026-01-28|随笔思考
|浏览量:

摘要：昨天我在“会计思维”和“指针本质”里找到了踏实感，甚至开始想象 PVE 和旧 i3 的实验田。可今天早上醒来，看着还没跑通的 C 代码，我第一次明确承认：这条路很硬核，也很长，而我现在最缺的不是热血，是时间。我需要先拿到工作这张入场券。于是我决定阶段性 All-in Python：用数据分析拿结果，用生态与岗位需求加速；更重要的是，Python 不只是“处理 Excel”的脚本，它同样通向 AI 和更大的工程世界。先活下来，再图发展。

时间： 2026.01.22
关键词： 求职优先 / 阶段性 All-in / 数据分析 / AI / 断舍离
状态： 焦虑后的通透：先把路走宽

🌅 1) 早晨的清醒：C 语言的“任重道远”

昨天我还在为“寄存器地址就是指针门牌号”兴奋，觉得自己终于摸到了一点底层的脉络。

但今天早上醒来，看着屏幕上还没有跑通的 C 代码，我心里突然涌上来一种深深的无力感。

我想做嵌入式，想玩 STM32，觉得控制硬件是男人的浪漫。

可不得不承认一个残酷的现实：C 语言这条路，太长、太陡了。

我不排斥底层技术，但我现在才刚起步。寄存器、内存管理、工程结构……要把它们练到能“靠它找到工作”的程度，可能需要几个月甚至更久。

而我现在等不起。

☀️ 2) 白天的抉择：先吃饭，再谈梦想

于是我做了一个决定：暂时战略性搁置嵌入式，全力切入 Python。

这不是放弃，而是“曲线救国”。

对当下的我来说，Python 有三个很现实的优势：

  • 上手快：能更快做出可展示的成果
  • 生态成熟：数据获取、清洗、报表、可视化都有现成轮子
  • 岗位需求明确：数据分析/自动化方向的“入场券”更容易够到

像今天我用 pandas 去抓数据、做报表，这种立竿见影的产出，正是我现在找工作最需要的敲门砖。

我必须先拿到那张入场券，把生活稳定下来。只有那样，我才有资本在未来的某个周末，气定神闲地重新拿起 STM32，继续硬件梦。

🌙 3) 晚上的惊喜：原来路变宽了

本以为转投 Python 只是为了“混口饭吃”的妥协，但到了晚上，随着研究的深入，我发现我错了。

当我从数据分析顺藤摸瓜摸到 AI 和大模型时，世界突然打开了。

Python 不仅仅是处理表格的脚本，它更像一把钥匙：

  • 一边连接着现实的饭碗（数据分析、自动化）
  • 一边连接着未来的核心（AI、工程化、算力）

看着哥推荐的云端算力平台，再看看那些用 Python 就能调用的 CUDA / 推理框架，我意识到：

这条路并没有我想象的那么“浅”。

🧳 4) 题外话：关于这台 Mac mini M4 的“断舍离”

说回这台陪我 emo 的 Mac mini M4。

买它的时候心气很高：Logic Pro 编曲、Final Cut Pro 剪辑、Fusion 360 建模，甚至还配了个 MIDI 键盘。

结果键盘找不到了，视频也没剪几个。

唯一让我觉得真香的还是 Fusion 360——比起我以前学的 3ds Max，它的时间轴能让我随时反悔修改，对新手太友好了。

但现在这台电脑终于不再只是“摆件”了：它变成了我的战场。

虽然它的 GPU 跑 AI 不如 N 卡，但用来写 Python、跑数据分析、做逻辑验证，它依然是把好手。

✅ 5) 今日总结：先活下来，再图发展

理想（C / 嵌入式）很丰满，现实（找工作）很骨感。

但 Python 像一座桥，让我能在解决温饱的同时，依然保留通往高塔（AI / 技术）的可能性。

先活下来，再图发展。

等我把工作稳定下来，我会回头把那条“任重道远”的路继续走完——只是顺序要对。
