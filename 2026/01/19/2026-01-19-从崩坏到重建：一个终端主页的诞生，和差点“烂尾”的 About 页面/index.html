我原本只是想写个简历，结果顺手重构了整个网站

发表于2026-01-19|更新于2026-01-28|技术日记
|浏览量:

摘要：我原本真的只是想写个简历。为了省事，我把“写简历”升级成“脚本生成”，又把脚本升级成“Web 应用”，再把它塞进自己的域名和 Hexo 体系里。然后站点开始 404/502 轮番报错，HTTPS 也跟着翻车。最后我砍掉复杂后端，回归纯静态，把网站救回来；修 Bug 的间隙，又顺手把主页做成了可输入的 Web 终端。

时间： 2026.01.19
关键词： 简历 / 过度设计 / 回归纯静态 / 终端主页
状态： 过山车式的一天，最终落地

现在是凌晨 12 点。看着屏幕上终于恢复的 HTTPS 绿锁，以及那个能响应键盘输入的终端主页，我长出了一口气。

如果非要用一句话总结今天，那就是：一个简历引发的“血案”。

1) 我真的只是想写个简历

距离上次更新过去两天了，今天原本的计划非常单纯：写一份新简历。

按照正常人的逻辑，打开 Word 或者 Markdown 写就完事了。

但我偏不。我觉得手写格式太麻烦，心想：

“既然我是搞技术的，为什么不写个脚本自动生成简历呢？”

2) 脚本变 Web 应用：需求原地升级

脚本写好了，跑起来了，但我又不满意了：命令行看着太枯燥，改内容还得动代码。

于是需求原地升级：

“我需要一个可视化的界面。”

就这样，我让 AI 帮我把脚本升级成了一个 Web 应用。

不仅能在线编辑，还能一键换肤、导出 PDF。看着那个功能越来越“像样”的编辑器，我非常有成就感。

3) 把“家”拆了：为了这口醋，差点把饺子包烂

看着这个刚出炉的 Web 应用，我又动了心思：这么好的东西，得挂到我的个人域名下才行。

于是，我试图把它集成进我的 Hexo 博客里，甚至还想搞个后端做在线 CMS。

灾难就在这一步发生了。

Nginx 配置冲突、HTTPS 证书报错、路径 404……原本运行良好的博客瞬间瘫痪。为了这口“醋”（简历工具），我差点把“饺子”（整个博客）给包烂了。

下午到晚上的大部分时间，我都在对着黑底白字的报错发呆：一遍遍重启服务，一遍遍看着浏览器转圈。

4) AI 救场：断舍离与回归纯静态

好在最后在 Cursor 的强力辅助下，我不仅修好了 Nginx，搞定了 SSL 证书，还把原本复杂的后端方案砍掉，回归了最稳健的纯静态。

当 HTTPS 绿锁重新亮起、页面不再 404/502 时，那种如释重负的感觉，比写出任何功能都爽。

这一刻我才意识到：复杂不是胜利，可维护才是。

5) 意外之喜：顺手造了一个“能打字的主页”

就在修 Bug 的间隙，我哥看着我的屏幕随口说了句：

“你主页那个光标只会闪，要是能打字就好了。”

我想着“反正都折腾一整天了，不差这一会儿”，顺口问了 Cursor 一句。

结果它真给我弄成了：现在我的主页就是一个全功能的终端模拟器，能响应键盘输入。

那是真的帅。

🧩 生活插曲

在这场高强度的技术“套娃”中，还穿插着生活的琐碎：

  • 帮侄子调好了尤克利利（顺便向 AI 学了指法）
  • 还抽空请教了 AI 一些金融知识

📝 总结

回过头看，今天真是神奇的一天：

写简历 → 写脚本 → 做 Web App → 集成域名/Hexo → 搞崩网站 → 修好网站 → 改成终端主页。

每一步都为了省事，结果每一步都在折腾。

但看着现在这个独一无二的网站，我觉得——值了。

晚安，明天继续。
