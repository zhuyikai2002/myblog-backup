时隔两月的更新：我把 Fedora 装进了 U 盘，打造专属随身工作站

发表于2026-03-21|更新于2026-03-21|技术日记折腾记录
|浏览量:

摘要：从 2026 年 1 月 28 日 到 2026 年 3 月 21 日，我鸽了将近两个月。不是跑路，而是把时间拿去做了一件更长期有价值的事：打造一个“插上就能干活”的 Linux 便携工作站。为了不影响主力机 Linux Mint，我把 Fedora Workstation 直接装进了 512GB 固态 U 盘，然后一步步补齐终端手感、软件安装链路、开发环境和 Git 免密发布。今天，这套系统终于从“能开机”进化到了“能生产”。

时间： 2026.03.21
关键词： Fedora / 便携系统 / dnf / Zsh / Git SSH / 回归更新
状态： 告别无效折腾，回归生产力

🌊 1) 潜水两个月后，为什么是现在回来

上一次更新还停在 1 月 28 日。一晃到今天 3 月 21 日，中间这段时间其实一直在折腾，只是没急着发。

这次我给自己定了一个目标：不再把主力机当实验田。
我的 Linux Mint 环境里已经有稳定工作流，没必要为了尝鲜新技术把它搞乱。

所以我决定另开一条线：做一个随插随用、独立可迁移、坏了也不拖累主力机的系统。

💽 2) 系统抉择：为什么是 Fedora，以及为什么装进 U 盘

我这次没有选双系统，也没有继续在虚拟机里“隔靴搔痒”。
而是直接把 Fedora Workstation 完整安装到一块 512GB 努米诺固态 U 盘，安装时选择 Use entire disk。

为什么选 Fedora：

  1. 想要更现代但克制的桌面体验，GNOME 默认审美正对胃口。
  2. 想跟上更积极的内核和工具链更新节奏。
  3. 底层足够纯粹，开发者生态成熟，适合长期打磨工作流。

避坑小贴士：这次我主动放弃了全盘加密

为了外置盘的读写性能和后续可维护性，我没有开 LUKS 全盘加密。
这样做的好处很实际：

  • 日常启动和读写更直接，尤其在 U 盘场景下体感明显。
  • 以后如果系统炸了，回主力机抢救数据更方便。
  • 排障路径更短，不会先卡在“解密链路”。

🧪 3) 注入灵魂：终端三件套把手感拉满

系统装好后，我第一件事不是换主题，不是搞 Dock，而是先把终端手感找回来。

我把默认 bash 换成了：

  • zsh
  • oh-my-zsh
  • zsh-syntax-highlighting
  • zsh-autosuggestions

装完后的差异非常直接：命令可读性和补全体验都明显上台阶。
写脚本、跑构建、查日志的那种“流动感”又回来了。

⚙️ 5) 打通生产力链路：VS Code、Git、SSH、Codex 一次到位

系统能“开机”和系统能“干活”，中间隔着一整套环境重建。
这部分我一口气做完了：

  1. 用命令行安装 VS Code 并接入官方源。
  2. 重新配置 Git 全局身份。
  3. 生成新的 ed25519 SSH 密钥并绑定 GitHub。
  4. 在 VS Code 里把 Codex 工作流配置好。
  5. 克隆博客仓库，恢复本地写作与部署链路。

当 git pull、git commit、git push 全部免密顺畅跑通后，那种熟悉的“任督二脉打通”感，真的很上头。

🚀 6) 这篇文章，就是新工作站交付后的第一次实战

这次更新本身，就是这套便携系统的验收单。

我在 Fedora U 盘系统里完成了：

  • 文章整理与润色
  • 仓库内容更新
  • 提交与推送
  • 自动部署发布

整个链路没有再出现“装好了但不想用”的尴尬。
它现在不是一个展示性质的玩具，而是已经变成了我的可移动生产环境。

✅ 总结：告别折腾感，进入可复用状态

以前很多折腾最后都会烂尾，不是因为技术做不到，而是因为链路不闭环。
这次我最满意的不是“装了 Fedora”，而是把它做成了能长期复用的系统资产。

从 2026 年 1 月 28 日 到 2026 年 3 月 21 日，这次停更背后并不是空白，而是一次底层工作流的重构。

现在可以很踏实地说一句：
我回来了，而且这次是带着一套更稳的生产力系统回来的。
